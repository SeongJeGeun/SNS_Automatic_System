# CODEBASE_DRIFT_PREVENTION.md

이 문서는 AI 에이전트와 개발자가 코드를 수정할 때 코드베이스가 다시 무질서해지는 현상(Drift)을 방지하고 생산성과 토큰 효율성을 영구히 방어하기 위한 서약서이자 체크리스트입니다.

## 🛠️ 개발 및 코드 변경 규칙
1. **소규모 분할 변경**: 코드 변경 시 한 번에 오직 1-2개의 파일만 수정하며, 수정 범위를 최소화합니다.
2. **선제적 QA 검증**: 코드를 수정한 후에는 즉시 리포지토리 루트에서 `./scripts/qa_checks.sh`를 실행하여 부작용을 검출합니다.
3. **즉각적인 롤백**: QA 검증 스크립트나 E2E 테스트가 실패하는 즉시, 변경 사항을 이전 안정 상태로 완전히 되돌립니다.
4. **정기 점검 수행**: 매 2주마다 본 문서의 체크리스트와 다음 점검 일정을 바탕으로 건강도를 점검합니다.
5. **데드 코드 즉시 박멸**: 사용되지 않는 임포트, 더 이상 구동되지 않는 임시 스크립트가 발견되면 즉시 아카이브 영역으로 밀어냅니다.
